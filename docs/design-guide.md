# TripPhoto デザインガイド

## 1. このガイドの目的

TripPhoto は、旅の写真を単なる一覧としてではなく、**地図上の移動の流れとともに振り返る体験**を提供するアプリである。

このガイドは、その体験価値を視覚・言葉・操作感のすべてで一貫して表現するための基準を定める。

本ガイドは、以下に適用する。

- iOSアプリ本体
- App Store掲載素材
- 公式Webページ
- バナー、フライヤー、紹介資料
- スクリーンショット
- 将来の関連プロモーション素材

本ガイドの目的は、見た目を統一することだけではない。  
TripPhoto に触れた人が、

- 旅に出たくなる
- 旅を残したくなる
- 写真を見返したくなる
- 場所と記憶が自然につながる

という感覚を持てるよう、体験の芯を一貫して表現することである。

---

## 2. ブランドの核

### 2.1 TripPhoto が扱うもの

TripPhoto が扱うのは、写真そのものだけではない。

写真が撮られた**場所**、そこへ向かう**移動**、そして旅全体の**流れ**を扱う。

TripPhoto は、旅の記録を保存するだけのものではなく、  
**あとから旅を思い出せるきっかけを、場所と流れの中に残しておくもの**である。

### 2.2 TripPhoto が提供する価値

TripPhoto は、旅の記憶を「点」ではなく「流れ」として取り戻す。

1枚ずつの写真を消費するのではなく、**どこを通り、どこで立ち止まり、どう旅が進んだか**を再体験できることが価値である。

その体験は、次のように説明できる。

- 写真から旅を思い出す
- 場所から記憶を呼び起こす
- 移動の流れから旅全体を感じ直す
- 断片的な記録を、ひとつの旅として再体験する

### 2.3 TripPhoto の立ち位置

TripPhoto は、単なる写真管理アプリでも、単なる地図アプリでもない。

**旅の写真体験を、地図の軌跡によって立体化するアプリ**である。

旅アプリらしさ、写真アプリらしさ、地図や軌跡の要素は重要だが、  
それらはすべて、**旅の記憶を流れとして再体験させること**に従属する。

## 3. デザイン思想

### 3.1 主役は「旅の流れ」

UIやビジュアルにおいて、主役は単独の写真ではなく、**写真が連なって旅になること**である。

そのため、孤立した一枚の強調よりも、**連続性、方向性、遷移、道筋**を大切にする。

### 3.2 情報は「整理」ではなく「再体験」に従う

写真整理アプリのような管理感を前面に出しすぎない。

TripPhoto は、整頓よりも、**見返したくなること**を重視する。

「一括管理」「効率化」「最適化」といった管理の言葉より、  
**思い出す、たどる、つながる、残す** という体験の言葉を優先する。

### 3.3 地図は背景ではなく、記憶の土台

地図は飾りではない。

写真を並べるための背景でもない。

旅の記憶を支える**構造そのもの**として扱う。

### 3.4 軌跡は説明ではなく、感覚を呼び戻す装置

軌跡は、経路情報の表示以上の意味を持つ。

それは「この順に進んだ」「ここへ向かった」という、旅の感覚をよみがえらせるための要素である。

## 6. ビジュアル原則

### 6.1 基本構造

TripPhoto のビジュアルは、以下の3層で考える。

- **地面 / 地図 / 現実の広がり**
- **軌跡 / 方向 / 流れ**
- **写真 / 思い出 / 断片**

この3層が分離されつつ、ひとつの体験としてまとまって見えることが重要である。

### 6.2 画面構成の考え方

画面は、情報を敷き詰めるのではなく、**旅の流れが自然に読める余白**を持たせる。

写真も地図も主張しすぎず、互いを引き立てる構成にする。

基本として次の関係を意識する。

- 上側には開放感
- 下側には安定感
- 視線には流れ
- 中心には写真または旅の記憶の核

### 6.3 軌跡表現

軌跡は、機械的な線ではなく、**移動した実感を持てる線**として扱う。

ただし装飾しすぎず、読みやすさを優先する。

### 6.4 写真表現

写真は旅の断片であり、主役である。

ただし TripPhoto においては、写真単体の美しさだけで完結させず、**位置や流れの中に置かれて初めて価値が立ち上がる**見せ方を優先する。

写真は説明素材ではなく、**記憶の入口**として扱う。

## 7. 色の考え方

### 7.1 目指す印象

色は、旅の高揚感を持ちつつも、落ち着きが必要である。

派手さより、**見返し続けられる穏やかさ**を優先する。

同時に、旅先の開放感、軽さ、空気感が感じられることも重要である。

### 7.2 色の役割

- ベース: 地図・地面・空気感を支える落ち着いた色
- アクセント: 軌跡や注目点を示す色
- サポート: 写真を邪魔せず、可読性を支える色

運用上の意味づけは次の通り。

- 青系: 空、広がり、清潔感、信頼感
- 緑系: 地面、場所、安心感、旅先の実在感
- オレンジ系: 移動、高揚感、旅の熱量
- 白・明色: 余白、整理、呼吸、構造の明快さ

### 7.3 避けるべき色運用

- 彩度の高い色の多用
- SNS的なネオン感
- ナビアプリの警告色に寄りすぎた配色
- 写真よりUIが目立つ配色
- 灰色中心で無機質にすること
- 多色を均等に使って焦点を失うこと
- くすみすぎて旅の高揚感が消えること

## 8. 形の考え方

### 8.1 基本姿勢

TripPhoto の形状は、**旅の流れを阻害しないこと**を優先する。

角張りすぎず、丸すぎず、情報の移り変わりに自然な形を採用する。

### 8.2 推奨される形

- 穏やかな曲線
- 滑らかな接続
- 写真と地図をつなぐ柔らかい境界
- 方向を示せる簡潔な矢印形状
- 緩やかな蛇行
- 開いた面
- 奥行きを感じる構図
- 地平線を感じる安定した水平基準

### 8.3 避けるべき形

- 強すぎる角
- 子ども向けアプリのような丸み過多
- 装飾目的の複雑なフレーム
- 情報の意味を持たない幾何学装飾
- 工業的で硬い直線ばかりの構成
- 機械的なグリッドだけで構成すること
- 意味のない複雑さ

## 9. タイポグラフィと言葉づかい

### 9.1 基本方針

言葉は、やさしく、簡潔で、押しつけない。

説明文は自然で、過剰にエモーショナルにしない。

文字は、旅の高揚感を壊さないように、明快で清潔に扱う。

### 9.2 フォント

#### 9.2.1 ブランドフォント

「TripPhoto」のアプリ名を大きく表示する場合は、**Plus Jakarta Sans**（Google Fonts）を使用する。

- フォント: `Plus Jakarta Sans`
- ウェイト: 800（ExtraBold）
- 用途: ヒーローセクション、バナー、プロモーション素材などでアプリ名を目立たせる場面
- 推奨サイズ: 72px（デスクトップ）、40px（モバイル 480px以下）
- レタースペーシング: -1px

読み込み:

```
https://fonts.googleapis.com/css2?family=Plus+Jakarta+Sans:wght@700;800&display=swap
```

#### 9.2.2 サブタイトルフォント

英語のサブタイトルやキャッチコピーには、**Caveat**（Google Fonts）を使用する。

手書き風の温かみがあり、Plus Jakarta Sans の構造的なアプリ名との対比で、旅の親しみやすさを表現する。

- フォント: `Caveat`
- ウェイト: 500（Medium）
- 用途: ヒーローセクションのサブタイトル、バナーのキャッチコピー
- 推奨サイズ: 36px
- アプリ名との間隔: margin-top -14px（TripPhoto に寄せて一体感を出す）

読み込み:

```
https://fonts.googleapis.com/css2?family=Caveat:wght@500&family=Plus+Jakarta+Sans:wght@700;800&display=swap
```

##### 使用上の注意

- Caveat は英語テキスト専用とする。日本語テキストには使用しない。
- 長文には使用しない。1文以内のキャッチコピーに限定する。
- Plus Jakarta Sans と組み合わせて使い、Caveat 単独でブランドを代表させない。

#### 9.2.3 タイトル・サブタイトルのサイズ比率定義

タイトル（Plus Jakarta Sans）とサブタイトル（Caveat）を組み合わせる際、すべてのサイズと間隔は**タイトルのフォントサイズを基準（1.0）とした比率**で決定する。

この比率を守ることで、フォントサイズを変えてもバランスが崩れない。

##### pt と px の換算

CSS における pt と px の関係は以下の通り。

- **1pt = 1.333px**（96 ÷ 72）
- **1px = 0.75pt**（72 ÷ 96）

pt は印刷由来の単位（1pt = 1/72インチ）、px はスクリーン由来の単位（CSS上 1px = 1/96インチ）である。
デザインツール（Illustrator、Keynote 等）では pt が標準、Web（CSS）では px が標準のため、換算が必要になる。

換算式:

```
px = pt × 1.333
pt = px × 0.75
```

##### 比率テーブル

| 要素 | 比率（対タイトル） | 54pt / 72px | 45pt / 60px | 30pt / 40px |
|---|---|---|---|---|
| タイトル（Plus Jakarta Sans 800） | **1.0** | 54pt / 72px | 45pt / 60px | 30pt / 40px |
| サブタイトル（Caveat 500） | **0.5** | 27pt / 36px | 22.5pt / 30px | 15pt / 20px |
| タイトル⇔サブタイトル間隔（margin-top） | **-0.194** | -10.5pt / -14px | -8.7pt / -12px | -5.8pt / -8px |
| ヒーロー上下パディング | **0.222** | 12pt / 16px | 10pt / 13px | 6.7pt / 9px |
| アイコン⇔テキスト間隔（gap） | **0.222** | 12pt / 16px | 10pt / 13px | 6.7pt / 9px |
| アイコンサイズ | **1.278** | 69pt / 92px | 57.5pt / 77px※ | 38.3pt / 52px※ |
| アイコン角丸 | **0.278** | 15pt / 20px | 12.5pt / 17px | 8.3pt / 12px |
| レタースペーシング（タイトル） | **-0.014** | -0.75pt / -1px | -0.6pt / -0.8px | -0.4pt / -0.6px |

※ アイコンサイズが92px（69pt）未満になる場合は、アイコンを省略する（9.2.5 参照）。

##### 適用ルール

- タイトルのフォントサイズが決まれば、他のすべての値は上記比率から算出する。
- サブタイトルは常にタイトルの半分のサイズとする。
- タイトルとサブタイトルの間隔は負の値（重なり方向）とし、一体感を出す。
- アイコン表示サイズの最小値（92px / 69pt）を下回る場合は、アイコンを省略する（9.2.5 参照）。
- デザインツールで pt 指定する場合は、上記テーブルの pt 値を使用する。CSSで実装する場合は px 値を使用する。

##### 実例

タイトルを45pt（60px）で使用する場合:

- サブタイトル: 45 × 0.5 = **22.5pt（30px）**
- タイトル⇔サブタイトル間隔: 45 × -0.194 = **-8.7pt（-12px）**
- 上下パディング: 45 × 0.222 = **10pt（13px）**
- アイコン⇔テキスト間隔: 45 × 0.222 = **10pt（13px）**
- アイコンサイズ: 45 × 1.278 = 57.5pt（77px） → **69pt（92px）未満のためアイコン省略**

#### 9.2.4 本文フォント

本文、説明文、UI テキストには、システムフォントスタックを使用する。

```
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;
```

#### 9.2.5 アイコン表示サイズの制約

TripPhoto のアプリアイコンは、**92×92px 未満では表示しない**。

92px の領域を確保できない場面では、アイコンを省略する。

## 10. 写真の扱い

### 10.1 写真選定方針

プロモーションで使う写真は、単に映えた一枚ではなく、**移動の途中にある感覚**があるものを優先する。

写真は、旅の事実を証明するためではなく、**記憶を開く入口**として選ぶ。

### 10.2 向いている写真

- 街を歩いた流れが想像できる写真
- 移動の途中で撮られた写真
- 複数地点の関係を感じられる写真
- 旅の時間経過が伝わる写真
- 空気感が伝わる写真
- 場所の魅力が感じられる写真
- 見返したくなる余韻がある写真

### 10.3 向いていない写真

- 位置性や旅感のないスタジオ的写真
- 被写体の強さだけで成立している写真
- 加工が強すぎる写真
- 旅行感ではなく商品訴求に見える写真
- 単なる記録証拠のような写真
- 情報過多で視線が散る写真

## 11. モーションの考え方

### 11.1 基本方針

動きは、驚かせるためではなく、**旅の流れを感じさせるため**に使う。

### 11.2 推奨されるモーション

- 滑らかな移動
- 写真と地図の自然な接続
- 軌跡をたどるような視線誘導
- 強すぎないフェードやスライド
- 手前から奥へ進む感覚
- 写真や要素の重なりによる自然な流れ

### 11.3 避けるべきモーション

- 派手なバウンス
- 過剰な拡大縮小
- ゲーム的な反応
- 速度で驚かせる演出
- 演出だけが前に出て、旅の流れを止める動き

## 12. UIデザイン原則

### 12.1 操作の意味が明確であること

TripPhoto は地図・写真・時間が関わるため、UI は特に明快である必要がある。

見た目の美しさより、**何が起きるかが自然に分かること**を優先する。

### 12.2 地図と写真の主従を固定しすぎない

場面によって主役は変わる。

一覧では旅全体、閲覧では一枚の写真、地図上では流れが主役になる。

それぞれの場面で重心を変えられる柔軟さを持つ。

### 12.3 旅の単位を尊重する

TripPhoto の中核は「旅を定義できること」である。

よって、旅の一覧、旅の区間、旅の切れ目は、UI上でも丁寧に扱う。

UI は次の優先順位で判断する。

1. 旅の体験として自然か
2. 写真が主役であることが伝わるか
3. 地図や軌跡の要素が体験に寄与しているか
4. 美しく、静かに読めるか
5. TripPhoto らしい独自性が出ているか

## 13. Web・バナー・ストア素材への展開指針

### 13.1 Web

- 冒頭で「旅」「写真」「地図」「軌跡」の関係を一言で伝える
- スクリーンショットは単なる機能一覧ではなく、旅の流れが感じられる順で並べる
- 派手なランディングページより、静かな信頼感を優先する
- 上部で開放感を出し、中央で旅の記憶の核を見せ、下部で機能を整理する
- セクションは単なる箱の積み重ねではなく、余白と導線で流れを作る

### 13.2 App Store スクリーンショット

- 1枚目で「何のアプリか」が一目で分かる
- 地図と写真が共存している画面を重視
- 軌跡や旅単位の価値が見える構成にする
- 「整理」より「振り返る」体験を主語にする
- 写真アプリ感だけ、地図アプリ感だけに寄らない

### 13.3 バナー

- 一言で価値が分かる
- 写真アプリ感だけにも地図アプリ感だけにも寄らない
- 旅の流れを感じる構図を使う
- 狭い面積では機能を全部見せるより、世界観を正しく伝えることを優先する
- 必要なのは「旅」「写真」「思い出」「つながり」のいずれかが短時間で伝わること

- 地図アプリっぽく寄せすぎる
- 写真整理アプリっぽく寄せすぎる
- 派手な旅行広告っぽくする
- 感情を言葉で過剰に説明する
- UIが写真や地図より目立つ状態を作る
- 管理ツールの言葉を前面に出す
- 情報を詰め込みすぎて旅の広がりを失う
- 無機質さや事務感を強める

---

## 15. TripPhoto のデザイン要約

TripPhoto のデザインは、

**旅の写真を、場所と移動の流れの中で見返す体験**を支えるためにある。

そのために必要なのは、

- 写真の美しさ
- 地図の構造性
- 軌跡の方向性
- 静かで誠実なトーン
- 再体験を促す見せ方
- 旅に出たくなる開放感
- 写真を見返したくなる余韻
- 場所と記憶が自然につながる構造

である。

TripPhoto は、旅の写真を並べるアプリではない。

**旅そのものを、地図の上でたどり直せるようにするアプリ**である。

## 16. ビジュアルシステム詳細

### 16.1 本章の目的

本章では、TripPhoto の世界観を一貫して再現するために、実制作で使用する具体的な視覚ルールを定める。

対象は以下とする。

- アプリアイコン
- アプリUI
- Webサイト
- App Store素材
- バナー
- フライヤー
- SNS投稿画像
- プレゼン資料

本章では、TripPhoto の象徴要素を次の4つに整理する。

- **空と広がり**
- **地平と大地**
- **軌跡と移動**
- **写真の記憶**

---

### 16.2 キービジュアル要素

TripPhoto の視覚表現は、以下の要素の組み合わせで成立する。

#### 16.2.1 空

空は、旅の開放感と軽さを表す。

青は単なる背景ではなく、**記憶が広がる空間**を表現する。

#### 16.2.2 地面

地面は、旅の現実感と移動の手触りを表す。

緑の面は、自然、平原、郊外、道中の広がりを抽象化したものである。

#### 16.2.3 軌跡

軌跡は、TripPhoto の最重要視覚要素である。

オレンジ系の流れは、単なる道路記号ではなく、**旅のエネルギーと進行方向**を示す。

#### 16.2.4 写真ピン

上部の写真を内包したピン形状は、「写真」と「場所」の結びつきを象徴する。

TripPhoto において写真は独立した画像ではなく、**地図上に置かれた記憶**である。

---

#### 16.3.1 ブランドカラー構成

---

### 16.4 グラデーションルール

#### 16.4.1 基本方針

TripPhoto のグラデーションは、立体感を誇張するためではなく、**空気感と移動感を出すため**に使う。

#### 16.4.2 推奨グラデーション方向

- 空: **上がやや濃く、下が明るい**
- 地面: **前景をやや濃く、奥を軽く**
- 軌跡: **進行方向に向かって明るくなる**
- ピン: **輪郭が少し重く、内側が軽い**

#### 16.4.3 禁止事項

- 金属的グラデーション
- 強いガラス感
- 彩度を上げすぎたネオン的グラデーション
- 3D押し出しに見える影

---

### 16.5 形状ルール

#### 16.5.1 地平線

地平線は、TripPhoto における重要な安定要素である。

**空と地面を明確に分ける水平基準**を持つこと。

ルール:

- 空と地面は混色しない
- 役割の異なる面は明確に分離する
- 地平線は、安定感を与える線として扱う

#### 16.5.2 軌跡形状

軌跡は、単なる道路の図記号ではなく、**旅の記憶の流れ**として扱う。

ルール:

- 線ではなく、**流れる帯状の面**として設計する
- 先端に向かって軽くなる
- 曲線は自然に流れ、急折れを避ける
- 一本道ではなく、旅の広がりを感じるS字や湾曲を許容する

#### 16.5.3 写真ピン形状

ルール:

- 角丸は穏やかに
- 吹き出しやピンの先端は、攻撃的にならない角度にする
- 写真の中身は、具体物より**記号性の高い風景**を優先する

---

### 16.6 線・境界・余白

#### 16.6.1 境界線

TripPhoto の境界は、分断ではなく整理のために存在する。

推奨:

- 白またはごく明るい面で境界を作る
- 仕切りは太すぎず、意味が伝わる太さにする
- 情報同士がぶつからないための抜けとして使う

#### 16.6.2 余白

余白は、旅の広がりを感じさせるために重要である。

ルール:

- 画面を情報で埋めすぎない
- 軌跡の周囲に呼吸できる余白を残す
- 写真・地図・説明文が互いに圧迫しない配置にする

#### 16.6.3 シャドウ

シャドウは極めて控えめに使用する。

推奨:

- 面の浮き上がりを最小限に補助する
- アイコンやカードの視認性補助に限定する

禁止:

- ドロップシャドウの多用
- UI全体が浮いて見える設計
- 強い外光表現

---

### 16.7 UIへの落とし込み

#### 16.7.1 主要CTA

- 主要ボタンは **Photo Pin Blue 系** を基本とする。
- 旅の核、選択、開始、閲覧導線に使う。

#### 16.7.2 補助CTA

- 補助的アクションには Sky Blue または中立色を使い、強い訴求が必要な場合のみ Route Orange を使う。

#### 16.7.3 軌跡表示

- 地図上のルートや移動方向表示には Route Orange 系を用いる。
- 必要に応じて時間帯・日別で若干色相をずらしてもよいが、
- ブランドの印象を崩さない範囲にとどめる。

#### 16.7.4 選択状態

- 選択・ハイライトは Photo Pin Blue を中心に構成する。
- エラーや削除系は赤に逃げすぎず、必要な場面だけ分離する。

### 16.8 Web・バナーでの展開ルール

#### 16.8.1 ヒーローセクション

トップページやキービジュアルでは、以下の構造を推奨する。

- 上部: 空 / 広がり
- 中央: 写真ピンまたは旅の記号
- 下部: 地面 / 軌跡
- 斜めまたは曲線: 旅の移動感

#### 16.8.2 ヒーローバナー実装仕様

公式Webサイトで使用しているヒーローバナーの具体的な仕様を以下に定める。
この仕様は、App Store素材、バナー、SNS画像、プレゼン資料などでヒーローを再現する際の基準とする。

##### レイアウト

- 構造: 横並び（flexbox）、中央揃え
- 左: アプリアイコン（92×92px、角丸20px）
- 右: テキストブロック（アプリ名 + サブタイトル）
- アイコンとテキストの間隔: 16px
- 上下パディング: 16px
- 左右パディング: 20px

##### 背景グラデーション

```
background: linear-gradient(135deg, #0078B8 0%, #78B800 100%);
```

- 方向: 左上から右下へ（135deg）
- 起点: Photo Pin Blue `#0078B8`
- 終点: Field Green `#78B800`
- 旅の移動感を斜めの方向で表現している

##### アプリ名「TripPhoto」

- フォント: Plus Jakarta Sans（Google Fonts）
- ウェイト: 800（ExtraBold）
- サイズ: 72px（デスクトップ）、40px（モバイル 480px以下）
- レタースペーシング: -1px
- 色: #FFFFFF

##### サブタイトル「Enjoy your photos on a map trail.」

- フォント: Caveat（Google Fonts）
- ウェイト: 500（Medium）
- サイズ: 36px
- 色: #FFFFFF（opacity: 0.92）
- アプリ名との間隔: margin-top -14px（TripPhoto に寄せて一体感を出す）

##### 再現時の注意

- アプリ名のフォントは必ず Plus Jakarta Sans を使用する。システムフォントで代替しない。
- アイコンは92px未満にしない。領域が足りない場合はアイコンを省略し、テキストのみ中央配置とする。
- グラデーションの方向（135deg）と色は変更しない。角度を変えると移動感の印象が変わる。
- サブタイトルは中核メッセージ（セクション5.1）またはその補助メッセージ（セクション5.2）から選択する。

##### 画像素材として書き出す場合の推奨サイズ

| 用途 | サイズ |
|---|---|
| Webバナー（横長） | 1200×300px |
| SNS OGP画像 | 1200×630px |
| App Store プロモーション | 1024×500px |
| スクエアバナー | 1080×1080px |

書き出し時は、上記レイアウトを維持しつつ、余白を広げて対応する。
背景グラデーションは素材全体に適用し、テキストとアイコンは中央に配置する。

#### 16.8.3 コピー配置

コピーは、視線の流れを妨げない位置に置く。

推奨:

- 上部中央
- 左上
- 軌跡の終点や起点を避けた安定位置

禁止:

- 軌跡を横切る配置
- ピンの主役性を崩す重ね方
- 長文を中央に密集させること

#### 16.8.4 スクリーンショット背景

スクリーンショットの背景や見出し帯には、

Sky Blue / White / Soft Gray を基本とする。

強いオレンジ背景は常用しない。

### 16.9 アイコン運用指針

#### 16.9.1 アイコンの意味

TripPhoto のアイコンは、

**写真・場所・旅の軌跡** を1枚に統合したブランドの要約である。

#### 16.9.2 アイコンで守るべき要素

- 上部の写真ピン
- 下部の地平線構造
- 地面と空の明確な分離
- オレンジ系の流れる軌跡
- 旅の進行方向を感じる上向きの抜け

#### 16.9.3 省略してよい要素

媒体サイズに応じて以下は簡略化可能。

- 雲の数
- 地面の段数
- 軌跡内の細かなグラデーション
- 写真内の風景ディテール

#### 16.9.4 省略してはいけない要素

- ピンと写真の関係
- 軌跡の存在
- 空と地面の分離
- 旅の方向性

### 16.10 実制作チェックリスト

制作物を公開する前に、以下を確認する。

- 旅の流れが見えるか
- 写真アプリ感だけに寄っていないか
- 地図アプリ感だけに寄っていないか
- 空と地面が混ざっていないか
- オレンジが強すぎて全体を壊していないか
- 青・緑・オレンジの役割が崩れていないか
- 余白が確保されているか
- 「整理」ではなく「再体験」の印象になっているか
- 旅に出たくなる空気感があるか
- 写真を見返したくなる余韻があるか
- 場所と記憶が自然につながって見えるか

---

### 16.11 本章の要約

TripPhoto の具体的なビジュアル運用は、

**青で空間を作り、緑で大地を支え、オレンジで旅の流れを示し、青い写真ピンで記憶を場所に結びつける**

という原則に基づく。

これにより、TripPhoto は単なる写真アプリでも地図アプリでもなく、

**旅の写真を地図の軌跡で楽しむ体験**として一貫した印象を保つ。
